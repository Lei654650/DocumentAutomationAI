# Document Automation AI V9.2

## AI 自动翻译配置

进入管理后台顶部的“AI 自动翻译设置”。

1. 在“翻译服务商”选择 OpenAI、DeepSeek、Google Gemini 或 Anthropic Claude。
2. 系统会自动填写推荐模型与接口地址。
3. 粘贴对应服务商的 API Key。
4. 点击“保存 AI 设置”。
5. 点击“测试连接”。
6. 测试成功后，选择需要翻译的订单并点击“开始自动处理”。

API Key 只保存在本机后端 `backend/data/ai_settings.json`，前端读取时只显示掩码。

## 默认配置

| 服务商 | 默认模型 | 默认接口 |
|---|---|---|
| OpenAI | gpt-4.1-mini | https://api.openai.com/v1 |
| DeepSeek | deepseek-chat | https://api.deepseek.com/v1 |
| Google Gemini | gemini-2.5-flash | https://generativelanguage.googleapis.com/v1beta |
| Anthropic Claude | claude-3-5-haiku-latest | https://api.anthropic.com/v1 |

模型名称可能随服务商更新而变化，服务商控制台显示的可用模型优先。

## V9.2 新增

- 服务商下拉列表由后端统一返回。
- 切换服务商自动填充默认模型与接口地址。
- 支持四种不同 API 协议。
- 保存后自动读取配置并显示 API Key 掩码。
- 连接测试显示翻译结果与耗时。
- 处理完成显示请求数、Token、耗时和预估 API 成本。
- API 错误会直接显示服务商返回的错误信息。
