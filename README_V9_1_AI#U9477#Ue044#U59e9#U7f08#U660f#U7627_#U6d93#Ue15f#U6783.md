# Document Automation AI V9.1 — AI 自动翻译

## 本版核心功能

管理员可在网页后台直接配置 OpenAI 或 DeepSeek，不再手动修改代码。配置成功后，订单点击“开始自动处理”会在后台异步执行：

1. 校验源文件
2. 分析文档结构
3. 调用 AI 翻译
4. 尽量保持 Office 文件结构与格式
5. 生成交付文件
6. 自动登记到订单“交付文件”区域

目前自动翻译输出：

- PPTX → 翻译后的 PPTX（文本框与表格，尽量保持对象样式）
- DOCX → 翻译后的 DOCX（段落与表格）
- XLSX → 翻译后的 XLSX（保留公式、工作表和主要格式）
- 可提取文字的 PDF → 翻译后的 DOCX
- 其他格式 → 暂时保留原文件进入人工审核

## 配置 AI 翻译

1. 启动软件并进入“订单后台”。
2. 在顶部“AI 自动翻译设置”选择 OpenAI 或 DeepSeek。
3. 填写 API Key。
4. 模型和接口地址可保留默认值。
5. 点击“保存 AI 设置”。
6. 点击“测试连接”。
7. 显示测试成功后，选择订单并点击“开始自动处理”。

API Key 只保存在本机：

`backend/data/ai_settings.json`

请勿把包含真实 API Key 的工程 ZIP 发送给其他人。

## 客户流程

首页上传文件 → 填写客户资料 → 选择输出与翻译要求 → 确认提交 → 保存订单号 → 查询进度 → 下载交付文件。

客户不会看到 API Key、内部处理日志和管理员备注。

## 管理员流程

进入订单后台 → 配置 AI 翻译 → 查看文档分析和建议报价 → 确认订单要求 → 开始自动处理 → 查看实时进度和日志 → 人工复核结果 → 完成交付。

## 注意事项

- AI 翻译会产生第三方模型费用。
- 大型文件包含的文本越多，处理时间越长。
- PPTX、DOCX、XLSX 可以尽量保留格式，但译文长度变化仍可能需要人工排版复核。
- 扫描 PDF 与图片需要另外安装 Tesseract OCR；本版重点为 AI 自动翻译。
